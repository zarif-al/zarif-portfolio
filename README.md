# Payload Next.js Template

A starter template for building content-driven websites with [Payload CMS 3](https://payloadcms.com) and [Next.js 16](https://nextjs.org). Ships with a page builder (blocks), rich text, SEO management, and a media library — everything wired up and ready to extend.

## Prerequisites

- **Node.js** ≥ 18.20.2 (≥ 20.9.0 recommended)
- **pnpm** ≥ 9
- **MongoDB** running locally or via the included Docker Compose stack

## Getting started

```bash
# 1. Clone and install
git clone <repo-url> my-project
cd my-project
pnpm install

# 2. Copy the environment file and fill in your values
cp .env.example .env

# 3. Start MongoDB (if not already running)
docker compose up -d mongo

# 4. Start the dev server
pnpm dev
```

Open [http://localhost:3000](http://localhost:3000) for the frontend and [http://localhost:3000/admin](http://localhost:3000/admin) for the Payload admin panel. Create your first user at `/admin` on first visit.

## Folder structure

```
src/
├── app/
│   ├── (frontend)/
│   │   ├── layout.tsx           <frontend root layout>
│   │   ├── [[...slug]]/         <dynamic CMS page routes>
│   │   ├── blog/[slug]/         <dynamic blog/post routes>
│   │   └── projects/[slug]/     <dynamic project routes>
│   ├── (payload)/
│   │   ├── layout.tsx           <payload admin layout>
│   │   └── admin/               <payload admin passthrough>
│
├── collections/                 <payload collection configs>
│
├── globals/                     <payload global configs>
│
├── blocks/                      <block configs used in layout fields>
│
├── fields/                      <reusable shared field groups>
│
├── tabs/                        <reusable admin tab configs>
│
├── hooks/                       <collection & global lifecycle hooks>
│
├── access/                      <access control functions>
│
├── lib/                         <framework-aware helpers & SDK wrappers>
│
├── utilities/                   <pure, stateless helper functions>
│
└── components/
    ├── admin/                   <custom admin UI components & overrides>
    ├── blocks/                  <react components, one per block type>
    │   └── RenderBlocks         <block type → component switcher>
    ├── layout/                  <header, footer, nav components>
    ├── ui/                      <shadcn generated components — edit directly, do not re-add without care>
    └── primitives/              <wrappers around ui/ with project defaults & variants>

payload.config.ts                <top-level payload config>
payload-types.ts                 <auto-generated types — do not edit>
```
